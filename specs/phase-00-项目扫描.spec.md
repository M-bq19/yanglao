# Phase 00 - 项目扫描

## 一、阶段名称

项目扫描 / 结构识别阶段

---

## 二、阶段目标

本阶段只做扫描与识别，不做正式代码修改。

目标是让后续美化有明确落点，避免盲改。

---

## 三、本阶段必须完成的任务

1. 扫描项目根目录结构
2. 确认 `back/` 与 `front/` 的职责边界
3. 找到后台公共布局文件
4. 找到后台菜单配置文件
5. 找到后台公共 CSS / 静态资源文件
6. 找到前台首页入口文件
7. 找到前台公共样式文件
8. 找到前台四类列表页文件
9. 找到前台搜索框、分页、卡片相关复用文件
10. 找到“疫情监测管理”和“物业收费管理”的菜单入口、页面入口、可见入口

---

## 四、本阶段禁止事项

本阶段原则上：

- 不做大规模代码修改
- 不做样式重写
- 不删除文件
- 不修改业务逻辑
- 不修改依赖
- 不修改数据库
- 不改接口结构

允许的唯一例外是：

- 为了后续记录需要，补充轻量注释或文档说明，但不是必须

---

## 五、本阶段输出内容

本阶段结束时，必须输出以下内容：

### 1. 项目关键文件清单

至少包括：

- 后台入口/公共布局文件
- 后台菜单配置文件
- 后台公共 CSS 文件
- 后台典型列表页文件
- 前台首页入口文件
- 前台公共样式文件
- 前台服务/物品/活动/资讯列表页文件
- 前台意见中心、个人中心相关文件

### 2. 文件职责说明

逐项说明：

- 这个文件是做什么的
- 为什么后续要改它
- 它属于公共层还是页面层

### 3. 风险判断

说明：

- 哪些文件可以放心改
- 哪些文件可能牵连业务逻辑
- 哪些文件暂时不应该动

### 4. 后续推荐修改顺序

必须明确给出：

1. 后台先改哪些文件
2. 前台再改哪些文件
3. 哪些页面最后再细化

---

## 六、本阶段验收标准

只有在以下条件都满足时，本阶段才算完成：

1. 已经识别出前后台关键文件
2. 已经找出公共布局、公共样式、菜单配置
3. 已经定位“疫情监测管理”“物业收费管理”的显示入口
4. 已经明确后续低风险修改顺序
5. 输出结果清晰可直接进入下一阶段

---

## 七、本阶段完成后的输出要求

当本阶段任务完成后，必须明确提示：

**本阶段已完成。**

并同时输出以下内容：

1. 本阶段已完成的事项
2. 本阶段修改的文件列表
3. 每个文件的修改摘要
4. 当前仍存在的风险或遗留问题
5. 是否满足进入下一阶段的条件

在完成上述输出后，再提示：

**建议进入下一阶段：**
`specs/对应的下一阶段文件名`

不得默认自动继续下一阶段，需等待确认。
