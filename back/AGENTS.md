# AGENTS.md

## 一、目录说明

当前目录为后端/后台相关工程目录：

`back/`

本目录本次任务的重点不是改 Java 业务，而是：

- 后台管理端页面整体美化
- 后台菜单优化
- 后台表格页优化
- 搜索区、分页区、按钮区的视觉整理
- 隐藏不再需要的后台功能入口

---

## 二、本目录本次任务核心目标

在不修改后端业务架构的前提下，完成后台管理端的统一视觉优化。

具体目标：

1. 去掉厚重黑边
2. 去掉深阴影
3. 去掉老式渐变
4. 左侧菜单改成扁平化浅色导航
5. 表格页改成白底卡片 + 浅灰分隔
6. 搜索区、表格区、分页区分层明确
7. “详情 / 修改 / 删除”改成轻量文本按钮或小圆角按钮
8. 去掉“疫情监测管理”和“物业收费管理”的后台入口

---

## 三、本目录禁止修改项

在 `back/` 目录中，以下内容默认禁止修改：

- Java 核心业务逻辑
- Controller 返回结构
- Service 逻辑
- Dao / Mapper SQL
- 数据库相关配置
- 登录认证逻辑
- Token 逻辑
- 拦截器逻辑
- 权限校验逻辑
- `pom.xml`
- 运行端口、上下文路径、数据库连接配置
- 任何影响 IDEA 启动方式的内容

---

## 四、本目录优先允许修改项

本目录中优先允许修改：

- 后台页面模板
- 后台静态资源
- 后台通用 CSS
- 后台菜单配置
- 后台相关静态 JS 中与样式和菜单显示有关的部分
- 后台布局类名（仅限样式需要）
- 页面容器结构中不影响业务的轻量调整

---

## 五、后台设计要求

### 5.1 菜单

左侧菜单需要：

- 扁平化
- 浅色背景
- 层级清晰
- 选中态明确
- 悬停态柔和
- 去掉厚重边框和压迫感阴影

### 5.2 表格页

后台列表页需要统一为：

- 白底卡片主容器
- 顶部搜索区单独成层
- 表格区单独成层
- 分页区单独成层
- 行分隔用浅灰
- 表头更简洁清晰
- 不再使用旧式重边框视觉

### 5.3 按钮

按钮统一要求：

- 1 个主色按钮
- 2 个辅助按钮风格
- 删除按钮弱化但仍可识别
- “详情 / 修改 / 删除”优先改成轻量文本按钮或小圆角按钮
- 统一圆角、内边距、字体大小

### 5.4 阴影和边框

统一要求：

- 只保留轻微阴影或不使用阴影
- 使用浅灰边框
- 去掉黑色粗边框
- 去掉老式立体按钮感

---

## 六、本目录范围裁剪要求

以下后台功能入口需要去掉或隐藏：

- 疫情监测管理
- 物业收费管理

处理顺序：

1. 先去菜单入口
2. 再去页面可见入口
3. 再排查是否有快捷入口或统计入口
4. 未经明确允许，不直接删除后端模块代码

---

## 七、本目录工作流程

在本目录工作时，必须按以下顺序：

1. 找到后台公共布局文件
2. 找到后台菜单配置文件
3. 找到后台公共 CSS 文件
4. 找到后台典型列表页模板
5. 先改公共框架
6. 再改典型列表页
7. 最后再做局部细节统一

---

## 八、本目录输出要求

每次修改前，先说明：

1. 预计修改哪些文件
2. 这些文件分别负责什么
3. 改动是否只影响页面层
4. 是否存在业务风险

每次修改后，必须输出：

1. 修改文件列表
2. 每个文件的修改内容
3. 是否涉及菜单隐藏
4. 是否涉及共享样式调整
5. 当前后台页面是否达到阶段目标

---

## 九、默认处理原则

若某项修改可能波及后端逻辑而不只是页面样式：

- 先停止
- 先说明风险
- 不直接越界修改

默认优先选择：

- 样式层解决
- 菜单显示层解决
- 模板结构轻量调整
- 公共样式统一

而不是：

- 改接口
- 改业务
- 改数据库
